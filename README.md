# ЭПЛ · Подпись CAdES-BES (detached)

Тестовый стенд (Vite + Vue3 + TS) для формирования **отделённой (detached) подписи
CAdES-BES** над XML-титулами ЭПЛ через **КриптоПро ЭЦП Browser plug-in**.

Формат подписи выбран под API оператора (Такском-Транспорт, раздел 4.14 «Отправление
подписанного титула»): оператор принимает `file` (XML «по формату» ФНС ЕД-7-26/116@) и
отдельный файл `sig` — открепленную подпись к байтам этого файла. XmlDSig/XAdES здесь
не используется.

## Предпосылки на машине

1. **КриптоПро CSP** (пробные 90 дней — бесплатно).
2. **КриптоПро ЭЦП Browser plug-in** + расширение в браузере.
3. Хотя бы один сертификат с закрытым ключом в хранилище **«Личное» (My)**.
   Тестовый ГОСТ-сертификат можно выпустить на https://www.cryptopro.ru/certsrv/.

## Запуск

```bash
npm install
npm run dev      # откроется http://localhost:5273
```

Проверка сборки/типов:

```bash
npm run build
```

## Как пользоваться

1. **Окружение** — страница сама проверит, что плагин активен, покажет версии.
2. **Сертификат** — «Загрузить сертификаты», выбрать подписанта.
3. **Данные** — вставить XML титула или загрузить `.xml`.
4. **Подпись** — получить `.sig` (DER-бинарь) и/или base64.

## Ключевые моменты реализации

- `src/useCryptoPro.ts` — работа с плагином: `checkEnvironment`, `getCertificates`,
  `signDetachedCadesBes`. Подпись: `CAdESCOM.CadesSignedData.SignCades(signer, CADES_BES, /*detached*/ true)`.
- `ContentEncoding = CADESCOM_BASE64_TO_BINARY` + `Content = base64(байты XML)` — хэш
  считается от **байтов**, а не от текста. Иначе оператор отклонит подпись.
- **Байт-в-байт**: подписывается ровно тот XML, что уйдёт оператору. Переформатировать
  после подписи нельзя.

## Дальше (не входит в этот стенд)

- Отдавать XML титула из `epl-integration` байт-в-байт (base64) и принимать `file + sig`,
  проверяя лишь наличие `sig`, затем слать в `/epl/add` Такскома.
- Серверная крипто-проверка не нужна — валидирует Такском/ГИС ЭПД.
